Section 01 / Thesis
Pair 01
Slide 02
From Forum Commons to Algorithmic Feeds

从论坛公共场，到算法信息流

推荐算法解决了信息过载，却把公共讨论变成了平台侧的注意力优化机器。
01 / The Old BBS Problem

线性时间线扛不住了

信息指数级增长，BBS 的线性时间线无法承载。检索、阅读、管理成本不断爆炸。好内容沉底，噪声淹没信号。
BBS 的开放讨论是好的，但信息架构扛不住规模。
02 / Algorithm's Contribution

它缓解了信息过载

推荐算法让用户更容易发现内容。它解决了「信息太多看不过来」的问题。个性化推荐提升了匹配效率。
推荐不是敌人 — 问题是「谁控制推荐」。
03 / Algorithm's Cost

平台黑箱与信息茧房

中心化分发 = 黑箱排序。注意力榨取 → 信息茧房 → 极化与割裂。用户失去过滤权，平台获得操纵权。
推荐算法把公共讨论变成了注意力优化机器。
Thesis: the trade-off we want to break.

Section 02 / Solution
Pair 02
Slide 03
Bring Recommendation Back to the User Side

让推荐算法回到用户侧

新 BBS 不是回到没有算法的时代，而是让算法从平台黑箱变成用户自己的信息代理。
01 / User-Side Agent

你的 Agent，你的过滤器

用户不再被平台算法投喂。Agent 代表用户阅读、筛选、总结、追踪上下文。过滤权从平台回归用户手中。
LLM 让每个人拥有自己的信息策展人。
02 / Staked Information Field

质押让信号有成本

完全免费的信息场域会被 spam 淹没。质押 USDC 发布帖子 = 经济信号。表达、任务、贡献都带有成本与责任。
经济约束代替平台审核。
03 / Web1 × Web2 × Web3 + Agent

三段进化

Web1 提供了开放讨论。Web2 证明了推荐/过滤的必要性。Web3 + Agent 让过滤权回到用户手里。
不是反推荐 → 是反平台侧黑箱推荐。
Solution: Agent + Staking = new information commons.

Cobo Agentic Wallet

Cobo Gives Agents Boundaries, Not Just Wallets

CAW 让 Agent 从聊天助手变成有边界的经济参与者。在我们的项目中，它不是「接了一个钱包」—— 它是让 Agent 在受约束、可审计的范围内执行链上操作的关键基础设施。
01 / Account Boundary

Agent 不拿私钥

Agent 在 CAW MPC 托管钱包的安全边界内行动。没有裸私钥风险，没有无限权限。钱包权限可隔离、可回收。Agent 操作的是受控账户，不是自己的钱包。
"Gives agents boundaries, not just wallets"
02 / Policy & Pact

每个任务都有可审查的授权

Pact 定义：intent、execution plan、policies、completion conditions。每笔 USDC approve 和 deposit 都有上限、目标合约、函数选择器、TX 次数、到期时间。人类在 CAW App 审批后才激活。
"CAW is the spending boundary"
03 / Auditable Execution

可追踪、可限制、可回收

Approve → deposit → post 每一步都有 request ID 和 tx hash 可追踪。Agent 的操作不是黑箱——Pact 可 revoke，权限可冻结。评审可以查看完整的 Policy Boundary 证据。
"Not black box ops, but scoped authority"
